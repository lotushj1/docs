## 為什麼要用 JSON？

傳統的文字提示詞雖然直觀，但當你需要快速調整配色、光線、構圖等參數時，往往要重寫整個提示詞。JSON 格式的結構化提示詞可以讓你：

<CardGroup cols={2}>
  <Card title="快速迭代" icon="arrows-rotate">
    只需修改特定參數，不用重寫整個提示詞
  </Card>
  <Card title="精確控制" icon="sliders">
    每個元素都有明確的欄位，避免遺漏
  </Card>
  <Card title="團隊協作" icon="users">
    不同角色可以修改不同欄位，互不衝突
  </Card>
  <Card title="批量產出" icon="copy">
    換參數就能快速生成系列作品
  </Card>
</CardGroup>

## 五大核心欄位一次搞懂

JSON 提示詞的標準結構包含五個核心欄位：

<AccordionGroup>
  <Accordion title="prompt（主要描述）" icon="comment">
    **用途**：描述圖片的主要內容和構圖
    
    ```json
    "prompt": "一位穿著藍色斗篷的女騎士，手持銀色長劍，站在古老城堡前"
    ```
    
    **重點**：要具體、生動，包含主體、動作、場景
  </Accordion>
  
  <Accordion title="style（視覺風格）" icon="palette">
    **用途**：定義整體的藝術風格和調性
    
    ```json
    "style": "奇幻插畫風格，溫暖色調，精緻細節"
    ```
    
    **重點**：風格要與 prompt 內容匹配
  </Accordion>
  
  <Accordion title="parameters（可調參數）" icon="gear">
    **用途**：存放需要頻繁調整的元素
    
    ```json
    "parameters": {
      "lighting": "溫暖的夕陽光線",
      "color_scheme": "藍色和金色",
      "composition": "三分法構圖",
      "mood": "英勇而神秘"
    }
    ```
    
    **重點**：這裡放最容易變動的元素
  </Accordion>
  
  <Accordion title="medium（媒介材質）" icon="image">
    **用途**：指定圖片的最終呈現方式
    
    ```json
    "medium": "數位插畫，高解析度"
    ```
    
    **重點**：影響最終的質感和細節
  </Accordion>
  
  <Accordion title="size（尺寸規格）" icon="expand">
    **用途**：定義圖片的尺寸和比例
    
    ```json
    "size": "16:9 橫向，適合網站橫幅"
    ```
    
    **重點**：根據使用場景選擇合適比例
  </Accordion>
</AccordionGroup>

## 超簡單範例：一鍵換色、換光線

### 基礎模板

```json
{
  "prompt": "一位優雅的女性，穿著白色長裙，站在花園中",
  "style": "攝影寫實風格，柔和光線",
  "parameters": {
    "lighting": "自然光線",
    "color_scheme": "白色和綠色",
    "mood": "清新自然",
    "background": "花園背景"
  },
  "medium": "高品質攝影",
  "size": "4:3 直向"
}
```

## 進階應用案例：遊戲角色介紹欄位

### 完整 JSON 模板

```json
{
  "prompt": "一位穿著藍色斗篷的女騎士，手持銀色長劍，站在古老城堡前",
  "style": "奇幻插畫風格，精緻細節",
  "parameters": {
    "character_class": "聖騎士",
    "weapon_type": "雙手劍",
    "armor_style": "重裝甲",
    "pose": "戰鬥姿態",
    "lighting": "神聖光線",
    "color_scheme": "藍色和銀色",
    "background_style": "中世紀城堡",
    "mood": "英勇而神聖",
    "art_style": "概念藝術風格"
  },
  "medium": "數位插畫，4K 解析度",
  "size": "16:9 橫向，適合遊戲宣傳"
}
```

### 快速變化範例

<Steps>
  <Step title="換職業">
    修改 `character_class`：聖騎士 → 法師 → 盜賊
  </Step>
  
  <Step title="換武器">
    修改 `weapon_type`：雙手劍 → 法杖 → 匕首
  </Step>
  
  <Step title="換場景">
    修改 `background_style`：城堡 → 森林 → 沙漠
  </Step>
  
  <Step title="換風格">
    修改 `art_style`：概念藝術 → 賽博龐克 → 蒸汽龐克
  </Step>
</Steps>

<Note>
**進階技巧**

這個 JSON 模板可以快速生成不同職業、不同場景的遊戲角色，只需要修改 `parameters` 中的對應欄位即可。
</Note>

## 還有哪些重點？

### JSON 提示詞最佳實踐

<AccordionGroup>
  <Accordion title="欄位層級一致" icon="align-left">
    **概念**：`prompt`、`style`、`parameters` 等應全在同一層；子欄位再往內縮排。
    
    **快速檢查方法**：觀察大括號 `{}` 是否對稱、縮排是否齊。
  </Accordion>
  
  <Accordion title="中英文標點一致" icon="language">
    **概念**：中文內容建議全形標點；英數與中文留半形空格，避免模型讀錯符號。
    
    **快速檢查方法**：Ctrl + F 搜尋半形逗號、句點混用。
  </Accordion>
  
  <Accordion title="使用易讀變數名" icon="tag">
    **概念**：`subject_type`、`bg_colors`… 越直白越好；別用 `x1`、`clrA` 等難以回憶的縮寫。
    
    **快速檢查方法**：「看變數可秒懂用途」為原則。
  </Accordion>
  
  <Accordion title="數值單位一致" icon="ruler">
    **概念**：角度統一用度數（°）、透明度用百分比等；避免有時用 0-1、有時用 0-100。
    
    **快速檢查方法**：在 `parameters` 加註單位，如 `15deg`、`80%`。
  </Accordion>
  
  <Accordion title="避免魔法字" icon="wand-magic-sparkles">
    **概念**：模型對「最高畫質」「Ultra HD」等詞反應不一，先測圖再決定是否保留。
    
    **快速檢查方法**：很多時是心理作用，實測有幫助再放。
  </Accordion>
  
  <Accordion title="可調元素放 parameters" icon="gear">
    **概念**：如果你想放進 Prompt 的東西是希望可以調整的，盡量就把他變成變數。
    
    **快速檢查方法**：一律放進 `parameters`（ex. `accessory_type`、`accessory_color`），方便後續調整。
  </Accordion>
</AccordionGroup>
